Uma livraria precisa de uma aplicação para controlar a venda de livros.

	REQUISITOS FUNCIONAIS

R1. O sistema deve manter informações cadastrais dos clientes e de seu histórico de vendas.

R2. O sistema deve permitir que o funcionário registre a venda de livros para clientes.

R3. O sistema deve manter informações do(s) funcionários que registram as vendas.

R4. O sistema de manter informações cadatrais sobre os livros.


	REQUISITOS NÃO FUNCIONAIS
	REGRAS DO NEGÓCIO

RN01. Um cliente não pode realizar uma compra caso o livro não tenha estoque.

RN02. Na compra de 3 livros ou mais, haverá um desconto de 10%.

RN03. Na compra de 6 livros ou mais, haverá um desconto de 15%.

RN04. Na compra de 10 livros ou mais, haverá um desconto de 25%


	Portabilidade
RN03. O sistema dever ser implementado com a linguagem de programação Java.

RN04. O sistema dever ser capaz de armazenar os dados em base de dados SQL Server ou SQLite.
